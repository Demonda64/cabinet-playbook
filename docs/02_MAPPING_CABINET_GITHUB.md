# Mapping Cabinet ↔ GitHub

## Cabinet (local)

- Dossiers internes = organisation + travail quotidien

## GitHub (règle simple)

- Tout ce qui est "client" = repo dédié privé
- Tout ce qui est "réutilisable" = repo dédié (public si clean)
- Tout ce qui est "process générique" = Cabinet (privé) ou Playbook (public)

## Décision rapide

- C’est client ? → repo `client-xxx` privé
- C’est réutilisable ? → repo `module-*` ou `template-*`
- C’est vitrine ? → repo `demo-*` public
- C’est administratif/légal ? → hors GitHub
