# Cabinet Playbook

Playbook public : méthodes, standards et checklists utilisés pour structurer un cabinet numérique (solo consultant).
Objectif : être lisible, auditable, durable.

## Ce repo contient

- Règles de structuration de repositories
- Conventions (commits, branches)
- Checklists (publier / archiver)
- Standards de sécurité et anonymisation
- Templates de README

## Ce repo ne contient pas

- Données client
- Documents légaux (contrats, factures, devis)
- Secrets (tokens, .env, clés)
- Infos commerciales sensibles
