# Workflow branches (solo)

## Principe
- `main` = stable, toujours “montrable”
- Branches courtes si besoin : `feat/*`, `fix/*`, `refactor/*`
- Merge rapide, suppression de la branche

## Règles simples
- Pas de GitFlow lourd
- Pas de release branches
- Pas de PR obligatoires (solo)

## Exemple
- `feat/add-audit-checklist`
- `fix/readme-typo`
