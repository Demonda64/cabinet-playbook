# Sécurité & anonymisation (public/privé)

## Public (OK)
- Templates neutres
- Démos avec données fictives
- Docs génériques
- Snippets techniques sans secrets

## Privé (obligatoire)
- Tout projet client
- Toute donnée identifiante (nom, email, tel, adresse)
- Tous éléments légaux (devis, contrats, factures)
- Accès, tokens, clés, exports, logs

## Anonymisation minimale
Avant publication :
- Remplacer noms, marques, emails, domaines par des placeholders
- Supprimer captures contenant identifiants
- Vérifier historique Git (aucun secret committé)
